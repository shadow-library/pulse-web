## Project Overview

Shadow Pulse (`pulse-web`) is a notification service UI for Shadow applications. It uses **TanStack Start** (SSR framework) with React 19, file-based routing via TanStack Router, and **Bun** as the package manager.

## Commands

```bash
bun dev              # Dev server on port 3000
bun run build        # Production build
bun run preview      # Preview production build
bun run test         # Run tests (vitest)
bun lint             # Lint (biome)
bun format           # Format (biome)
bun check            # Full biome check (lint + format)
```

## Git Hooks

- **pre-commit**: runs `bun lint` and `bun run test`
- **commit-msg**: enforces Conventional Commits via commitlint

## Architecture

### Routing (TanStack Start + Router)

File-based routing under `src/routes/`. Routes define config, loaders, and page composition only — no components or business logic. The router is configured in `src/router.tsx` with scroll restoration and intent-based preloading. `src/routeTree.gen.ts` is auto-generated; never edit it.

### Code Organization

See `docs/PROJECT_STRUCTURE.md` for the full guide. Key layout:

- **`src/routes/`** — Route definitions only (use `createFileRoute`/`createRootRoute`)
- **`src/views/`** — Page-private code (components, hooks, helpers, styles). Mirrors `routes/` structure. Opt-in: only created when a route needs extracted components/hooks. Uses `_self/` when a route has children.
- **`src/features/`** — Shared business logic grouped by domain
- **`src/components/`** — Shared, domain-agnostic UI components
- **`src/hooks/`** — Shared, domain-agnostic hooks (kebab-case filenames)
- **`src/lib/`** — API clients, utilities, config. `api.gen.ts` is auto-generated from OpenAPI spec.

### UI Layer

Uses `@shadow-library/ui` (wraps Mantine 8) and `lucide-react` for icons. Scoped styles use CSS modules (`*.module.css`).

## Code Conventions

### Hard Rules

- **1 component per file, 1 hook per file** — no exceptions
- **Named exports only** — no default exports anywhere
- **`*.gen.ts` files are never edited manually** — generated by tooling
- **Promote when shared** — if 2+ pages use the same thing, move it to shared folders

### File Section Headers

Every `.ts`/`.tsx` file uses this comment structure to organize sections:

```ts
/** Importing npm packages */
/** Importing user defined modules */
/** Declaring types */
/** Declaring constants */
```

### Formatting (Biome)

- Single quotes, trailing commas, always semicolons
- JSX single quotes, arrow parens only when needed
- 2-space indent, 180 char line width

### TypeScript

- Strict mode with all strict checks enabled plus `noUncheckedIndexedAccess`
- Path alias: `@/*` maps to `src/*`
- `verbatimModuleSyntax` is on — use `import type` for type-only imports
